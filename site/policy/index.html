Privacy Policy

Transmux is built as a local-first desktop media converter. The app does not have developer-run accounts, a backend service, advertising, or hidden telemetry.

Last updated: May 5, 2026 App telemetry: None Third-party services: Optional

01

Scope

This Privacy Policy applies to the Transmux desktop application, the Transmux website, release downloads, and related support channels operated by Piyush Doorwar (the "Developer"). By using Transmux or the Transmux website, you agree to this policy.

02

Plain-English Summary

Transmux converts local media files on your device. The Developer does not collect your media files, conversion history, subtitle files, settings, or activity from the app.

No app accounts. No developer backend. No hidden app telemetry.

03

Data the App Stores Locally

Transmux stores limited data on your device to preserve your preferences between sessions:

  • Your last selected output format.

On supported desktop systems this data is stored in the operating system's application data area, including a Transmux settings.json file. It remains on your device unless you share it, back it up, sync it, or delete it through your system.

04

Media Files and Local Conversion

Transmux opens input files you choose through the file picker or drag and drop, and writes output files to a path you specify. Conversion is entirely local. The app does not upload your audio, video, subtitle files, or converted output to the Developer.

If you use operating-system integrations, installers, file associations, sandbox permissions, or Flatpak portals, your operating system may process file paths and permissions according to its own behavior.

05

Update Checks

The About dialog includes a manual "Check for Updates" action. When you use it, Transmux contacts the GitHub releases API to check the latest published version. GitHub may receive your IP address, user agent, request time, and other standard request metadata.

The update check is not routed through a Developer-controlled server. Transmux does not send your media files, settings, or personal files as part of the update check.

06

Website and Release Downloads

The Transmux website fetches public release information from GitHub so it can show download links. When you visit the website, download releases, open GitHub links, or report issues on GitHub, GitHub and your browser/network providers may receive standard web request data.

The Transmux website currently uses Cloudflare Web Analytics. Cloudflare may process limited website analytics data such as page views, referrers, device/browser information, approximate location derived from IP address, and similar request metadata. The Developer does not use the Transmux desktop app to send analytics events.

07

Support, Email, and Issues

If you email support, open a GitHub issue, submit a pull request, or otherwise contact the Developer, you choose what information to provide. That may include your name, email address, GitHub username, operating system, logs, screenshots, media filenames, file paths, or other diagnostic details.

Please avoid sending private media files, sensitive screenshots, secrets, passwords, API keys, or personal information that is not needed to handle your request.

08

What the Developer Does Not Do

  • The Developer does not sell your personal data.
  • The Developer does not operate Transmux user accounts.
  • The Developer does not collect your media files from the app.
  • The Developer does not track your conversions or file activity from the app.
  • The Developer does not receive your local media files, subtitle files, or conversion output from the app.
  • The Developer does not use the desktop app for advertising, profiling, or hidden telemetry.
