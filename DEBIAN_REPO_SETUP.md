# Debian Repository Setup

This document explains how the Debian repository is set up and maintained for Transmux.

## How It Works

1. **Automatic Publishing**: When you tag a release (e.g., `v1.1.0`), GitHub Actions automatically:
   - Builds the `.deb` package
   - Generates Debian repository metadata using `reprepro`
   - Commits changes to `site/debian/`
   - GitHub Pages deploys the updated site

2. **User Installation**: Users can add the repository:
   ```bash
   echo "deb https://piyushdoorwar.github.io/transmux/debian stable main" | sudo tee /etc/apt/sources.list.d/transmux.list
   sudo apt update
   sudo apt install transmux
   ```

## Directory Structure

```
site/debian/
├── conf/
│   └── distributions     # Reprepro configuration (created by GitHub Actions)
├── dists/
│   └── stable/          # Debian repo metadata (auto-generated by reprepro)
├── pool/
│   └── main/            # .deb packages
└── indices/             # Repository indices (auto-generated)
```

## Workflow Details

The **publish-debian-repo** job in `.github/workflows/release.yml`:
- Triggered by every tag push (e.g., `v1.1.0`)
- Downloads the built `.deb` from the build job
- Uses `reprepro` to add it to the repository
- Commits and pushes changes back to `main` branch
- GitHub Pages auto-deploys the updated `site/` folder

## Managing the Repository

### Adding a new release
Just push a tag – the workflow handles everything:
```bash
git tag v1.1.0
git push origin v1.1.0
```

### Removing a package
To remove a specific version from the repository:
```bash
cd site/debian
reprepro remove stable transmux
```

Then commit and push:
```bash
git add site/debian/
git commit -m "Remove transmux from Debian repository"
git push
```

### Local Testing

To test the repository setup locally:
```bash
# Build the .deb
./scripts/build-linux.sh

# Set up repo structure
mkdir -p site/debian/{conf,dists/stable/main/binary-amd64,pool/main}

# Create distributions file
cat > site/debian/conf/distributions <<'EOF'
Codename: stable
Components: main
Architectures: amd64
Description: Transmux media converter
EOF

# Add the .deb
cd site/debian
reprepro includedeb stable ../../artifacts/packages/*.deb
```

## GitHub Actions Configuration

Requires:

- Tag-based releases (existing workflow)
- GitHub Pages enabled on main branch (already configured)
